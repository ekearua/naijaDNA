naijaDNA Streaming Budget and Platform Recommendation

This addendum supplements the corrected scaling report with streaming-specific budget guidance and a platform recommendation matched to the current codebase.

Date: 2026-04-07 Current stack: LiveKit already integrated in backend and Flutter Primary question: what platform should power streams as usage grows?
The current codebase already uses LiveKit for stream room tokens and media sessions. That means keeping LiveKit is the lowest-change path. Switching to a broadcast-first platform like Mux becomes attractive only when your audience pattern shifts toward large passive viewing.

Platform Recommendation

Recommended now: Keep LiveKit

For the current codebase, LiveKit is the right platform to keep using for streaming because:

  • it is already integrated in both backend and Flutter
  • the current stream feature is interactive, not just passive broadcast
  • changing providers now would create avoidable backend and client rewrite work
  • LiveKit already fits the current room-token architecture and viewer join flow

Recommended later: add Mux if streams become broadcast-heavy

If your product evolves into large one-to-many broadcast events, the best medium-term architecture is likely:

  • LiveKit for host, backstage, or guest coordination
  • Mux for large-scale viewer delivery
  • Keep the rest of the application flow unchanged
Recommendation in one line: keep LiveKit now, because it matches the existing implementation; only move to Mux or a LiveKit plus Mux hybrid when the audience becomes mostly passive and stream viewer counts materially outgrow the economics of interactive WebRTC delivery.

Streaming Rollout Path

  1. Continue with LiveKit for the current interactive streaming feature.
  2. Measure actual monthly stream minutes, concurrency, and average viewer counts.
  3. If streams stay interactive, scale within LiveKit.
  4. If broadcasts become large and mostly passive, introduce Mux for viewer delivery while keeping LiveKit for host-side interactivity.
